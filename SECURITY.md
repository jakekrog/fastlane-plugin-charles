# Security Policy

## Supported Versions

Only the latest published version of `fastlane-plugin-charles` is supported with security updates.

## Reporting a Vulnerability

Please **do not** open a public issue for security vulnerabilities. Instead, report it privately via a [GitHub Security Advisory](https://github.com/jakekrog/fastlane-plugin-charles/security/advisories/new) for this repository.

Include as much detail as you can: the affected version, a description of the vulnerability, and steps to reproduce it if possible. This is the same private-reporting channel referenced in the [Code of Conduct](CODE_OF_CONDUCT.md).
